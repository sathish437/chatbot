## Prerequisites
- Node.js installed on your Windows machine.
- A valid **Google Gemini API Key**.

## Quick Start Commands (Windows PowerShell or CMD)

### 1. Setup Backend
Open a terminal in the `server` folder:
```powershell
cd server
npm install
```

**Crucial Step:** Open the `server/.env` file and replace `YOUR_GEMINI_API_KEY_HERE` with your actual Google AI Studio key:
```
GEMINI_API_KEY=AIzaSy...
```

Then start the server:
```powershell
npm run dev
```
*Server runs on: http://localhost:5000 (Uses Nodemon to auto-restart on changes)*

### 2. Setup Frontend
Open a **new** terminal in the `client` folder:
```powershell
cd client
npm install
npm run dev
```
*Client runs on: http://localhost:5173*

## Testing the Chatbot

Open your browser to `http://localhost:5173`.

### Example Queries to Try:
1. "Hello, how are you?"
2. "I need to reset my password" (Triggers Knowledge.json context)
3. "What are your business hours?" (Triggers Knowledge.json context)
4. "Can you tell me a joke about coding?" (Uses Gemini's general knowledge)
5. "Can you summarize our conversation?" (Tests chat memory)

### Features to Verify:
- **AI Intelligence**: Responses are generated by `gemini-2.5-flash`.
- **Dynamic Context**: The AI strictly uses `knowledge.json` for specific company facts.
- **Persistence**: Refresh the page – chat history remains via LocalStorage.
- **Graceful Errors**: If the API key is missing, it responds with a helpful offline message instead of crashing.

## Troubleshooting
- **Bot returns a generic Error Message in chat window**: This means your `GEMINI_API_KEY` in `server/.env` is missing or invalid. Check your Google AI Studio account!
- **Server address already in use (EADDRINUSE)**: Another node process is stuck in the background. Kill it using `taskkill` or stop the running terminal.
- **Frontend says 500 API Error**: Check the backend terminal running `npm run dev` to see the exact crash log.
